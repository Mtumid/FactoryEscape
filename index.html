⚡ Enemies
ENEMY_SPEED
// Pixels per frame each enemy moves on their patrol. 0 = all enemies stop completely. 3 = very fast and scary! The code multiplies this by their direction (1 or -1).
1.2
ENEMY_HIT_RADIUS
// How close (in pixels) Runner must get to an enemy to lose a life. Bigger = enemies feel huge and unfair. Smaller = you can slip past them. Default is 22px.
22
LIVES
// How many lives you start with. This is an integer (whole number). The game reads this once at the start — so Apply then press Play Again to see the change.
5

FIREBALL_DURATION
// Seconds you can shoot fireballs. Internally stored as seconds × 60 frames. Jump while 🔥 is active to shoot!
8

// Pixels per frame the fireball travels. 8 = default. 20 = laser-fast. 3 = floats slowly across the screen. Watch it hit enemies at different speeds!

// Seconds all enemies stay frozen (bright blue) after you collect ❄️. The code sets each enemy's "frozen" counter to this × 60 frames.

// Seconds of invincibility after collecting ⚡. Runner flickers and rainbow orbs orbit him — the flickering is the code skipping drawing every 5th frame!

🧪 Advanced
CAMERA_LAG
// How smoothly the camera follows Runner. 0.1 = smooth and floaty (default). 1.0 = instant snap — no lag at all. This is called "lerping" — the camera travels this fraction of the gap each frame.
0.1
SQUISH_AMOUNT
// How much Runner squashes on landing. 1.0 = no squish (boring!). 0.5 = very squashed (funny!). This is "squash-and-stretch" — one of the 12 principles of animation.
0.6
PARTICLE_COUNT
// How many sparkle particles appear when Runner gets hit. 12 = default. 50 = explosion! 1 = one lonely dot. Particles are tiny objects the code creates and then deletes after 35 frames.
12
SCORE_PER_FIREBALL
// Points awarded when a fireball hits an enemy. 50 = default. Try 1 or 999! In code: score += SCORE_PER_FIREBALL. The += means "add this to what's already there".
50
